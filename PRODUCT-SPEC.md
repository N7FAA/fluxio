# FFmpeg 格式转换工具 · 产品规格与架构文档

> 面向桌面端的产品梳理、架构重设计及界面重设计参考。目标：体验好、界面质量高，并为后续打包成桌面应用做准备。

---

## 一、产品定位与目标

### 1.1 产品是什么

- **一句话**：一款本地优先、界面清晰的**视频/动图/图标格式转换**桌面工具，支持队列、预设、画质对比与 PNG→ICNS。
- **核心价值**：在「简单可用」与「参数可控」之间取得平衡——普通用户能快速完成常见转换，进阶用户能细调编码与画质。

### 1.2 目标用户

| 用户类型     | 需求简述                     | 产品侧重点           |
|--------------|------------------------------|----------------------|
| 普通用户     | 转格式、压体积、做动图/图标  | 预设、拖拽、进度可见 |
| 创作者/UP 主 | 画质可控、批量、对比效果     | 队列、对比、预设     |
| 开发者       | 自动化、可脚本、可集成       | 参数透明、日志、API  |

### 1.3 产品目标（重设计后）

- **体验**：主流程 3 步内完成（选文件 → 选目标/预设 → 开始），高级选项不干扰主流程。
- **界面**：信息层级清晰、视觉统一、符合桌面应用习惯（含后续 Electron 打包）。
- **架构**：前后端职责清晰，便于后续接入 Electron、多窗口、系统托盘等。

## 二、当前架构简要分析

### 2.1 技术架构（现状）

```
┌─────────────────────────────────────────────────────────┐
│  Browser (单页)                                          │
│  · index.html 单页，所有功能平铺在同一长页               │
│  · app.js 单体脚本：表单、队列、预设、对比、ICNS          │
│  · styles.css 全局样式                                  │
└───────────────────────────┬─────────────────────────────┘
                            │ HTTP / REST
┌───────────────────────────▼─────────────────────────────┐
│  Node.js (Express)                                       │
│  · server.js：路由、队列、FFmpeg/ffprobe/iconutil 调用   │
│  · 上传、预览、流式播放、预设文件读写                     │
└───────────────────────────┬─────────────────────────────┘
                            │ 子进程 / 文件系统
┌───────────────────────────▼─────────────────────────────┐
│  FFmpeg / ffprobe / iconutil (系统或内置)                │
└─────────────────────────────────────────────────────────┘
```

### 2.2 信息架构（现状）

- **单页多区块**：输入输出、视频参数、音频参数、自定义参数、预设管理、预览、视频对比、图标转换(ICNS)、任务队列 等全部纵向堆叠。
- **问题**：  
  - 主流程（选文件→选格式→转换）与高级功能、工具型功能混在一起，首屏重点不突出。  
  - 无「场景入口」（如：我要转视频 / 做动图 / 做图标），用户需在长页中自己找。  
  - 队列与任务反馈分散，缺少统一的「工作区」感受。

### 2.3 交互与视觉（现状）

- 以表单为主，按钮、输入框、下拉框样式统一但层级弱。  
- 缺少明确的视觉焦点（如：当前步骤、主操作按钮）。  
- 无拖拽添加文件、无缩略图/卡片化文件列表。  
- 暗色主题已存在，但缺少系统化的间距、圆角、字阶规范。

## 三、推荐竞品（体验好、界面质量高）

以下为同类型或相近类型工具，**重点看交互与界面**，可作为重设计参考。

### 3.1 Permute 3（Mac）

- **定位**：Mac 端视频/音频/图像格式转换，商业软件。  
- **体验与界面**：  
  - 以**拖拽**为主要输入方式，图标/缩略图展示待转换文件。  
  - **目标格式**用大按钮或清晰列表选择，设备/平台预设（如 iPhone、Apple TV）一目了然。  
  - 界面极简，高级选项折叠或放在二级界面，不打扰主流程。  
  - 队列与进度条清晰，批量时逐个展示状态。  
- **可借鉴**：拖拽优先、预设前置、主界面只保留「文件 + 目标格式 + 开始」三条主线。

**官网**：[Permute 3](https://apps.apple.com/us/app/permute-3/id1444998321)（Mac App Store）

---

### 3.2 HandBrake（跨平台）

- **定位**：开源视频转码器，基于 FFmpeg，跨平台。  
- **体验与界面**：  
  - 2024 年 1.8 升级 GTK4，界面更统一；Windows/macOS 有原生控件。  
  - 预设（General、Web、设备等）放在显眼位置，常用场景一键选择。  
  - 视频/音频/字幕用**标签页**分区，结构清晰。  
  - 预览窗口、队列与日志分离但可及。  
- **可借鉴**：用「预设/场景」作为第一层，再进入「详细参数」；标签页分区避免单页过长。

**官网**：[handbrake.fr](https://handbrake.fr/)

---

### 3.3 Shutter Encoder（Windows / macOS / Linux）

- **定位**：面向剪辑/出版的免费工具，FFmpeg 内核，功能多。  
- **体验与界面**：  
  - 左侧**功能列表**（转码、压缩、提取音频等）按任务分类，选功能即进入对应参数区。  
  - 中间文件列表，右侧参数与预览，布局清晰。  
  - 为「专业用户」设计，但通过分类和默认值降低门槛。  
- **可借鉴**：按「任务类型」划分入口（转码 / 压缩 / 动图 / 图标），而不是按「参数类型」（视频/音频/自定义）划分。

**官网**：搜索 "Shutter Encoder" 即可找到官网下载。

---

### 3.4 Adapter（Mac / Windows）

- **定位**：免费，支持视频/音频/图片转换，集成 FFmpeg + VLC。  
- **体验与界面**：  
  - 简洁，下拉选格式即可，高级选项折叠。  
  - 支持**并排预览**、批量、简单剪辑。  
- **可借鉴**：格式选择用下拉+图标或卡片，减少纯文字列表；预览与参数并排。

**官网**：[macroplant.com/adapter](https://macroplant.com/adapter)

---

### 3.5 Expi（Mac / Windows）

- **定位**：FFmpeg GUI 桌面应用，一次购买。  
- **体验与界面**：  
  - **拖拽**添加文件，支持 200+ 格式，本地处理、无大小限制。  
  - 界面偏简洁，强调批量与硬件加速。  
- **可借鉴**：强调「拖拽 + 格式卡片」的添加方式，弱化路径输入。

**官网**：[getexpi.app](https://getexpi.app/)

---

### 3.6 Media Tool（Mac）

- **定位**：Mac 端「全媒体库转码」工具，强调简洁与最佳实践。  
- **体验与界面**：  
  - 统一处理视频/图/音频，性能与隐私（本地）并重。  
  - 设计语言偏现代 Mac 应用。  
- **可借鉴**：统一入口（一个「媒体转换」概念），再按类型或预设分流。

## 四、建议的产品架构（重设计后）

### 4.1 信息架构（IA）

- **第一层：场景入口（主导航）**  
  - **转视频**：视频/音频格式转换，预设 + 自定义参数。  
  - **动图**：GIF / WebP / APNG，预设（高质量/小体积等）。  
  - **图标**：PNG → ICNS，可选多尺寸导出。  
  - **画质对比**：双视频并排播放与参数对比。  
  - **设置**：根目录、最大文件大小、主题等（桌面版可含「开机自启」等）。

- **第二层：各场景内**  
  - **转视频**：文件区 → 目标格式/预设 → 参数（可折叠）→ 队列。  
  - **动图**：文件区 → 动图预设 → 可选参数 → 队列。  
  - **图标**：文件区 → 输出选项（单 ICNS 或多尺寸）→ 执行。  
  - **画质对比**：文件 A/B 选择 → 并排播放区 + 参数信息。

- **第三层：全局**  
  - **队列**：所有任务的统一队列（转码、动图、图标），可放在侧栏或底部栏，始终可及。  
  - **预设管理**：在「转视频」或「设置」下，统一管理、编辑、删除预设。

这样「产品架构」清晰：**按场景分模块，按流程分步骤，队列与设置全局可见**。

### 4.2 技术架构（为桌面端预留）

- **前端**：  
  - 单页应用（SPA）即可，但按「场景」拆成**视图/路由**（如 `/convert`、`/gif`、`/icns`、`/compare`、`/settings`），便于后续 Electron 多窗口或深链接。  
  - 组件化：文件选择区、格式/预设选择器、参数表单、队列列表、对比播放区等可拆成独立组件，便于复用与测试。

- **后端**：  
  - 保持现有 Express + 队列 + FFmpeg/iconutil 的职责；  
  - API 按「资源」划分更清晰：`/api/jobs`、`/api/presets`、`/api/convert`、`/api/compare`、`/api/icns` 等，方便桌面端用同一套 API。

- **桌面端（后续 Electron）**：  
  - 主进程：启动 Express（或内嵌）、管理窗口、托盘、菜单。  
  - 渲染进程：加载当前 Web 前端（本地或 localhost），避免两套 UI。

### 4.3 用户主流程（重设计后）

1. **打开应用** → 看到「场景入口」（转视频 / 动图 / 图标 / 对比），而不是一屏长表单。  
2. **选场景** → 进入该场景页：左侧或顶部为「添加文件」（拖拽或选择），中间为「目标格式/预设」，右侧或下方为「高级参数（可折叠）」。  
3. **添加任务** → 点击「加入队列」或「开始」，任务进入**全局队列**；队列支持进度、取消、打开输出目录。  
4. **其他**：画质对比、预设管理、ICNS 作为独立入口，流程各自闭环，但队列可统一。

## 五、界面重设计方向

### 5.1 布局建议

- **整体**：  
  - **顶部**：Logo + 场景导航（转视频 | 动图 | 图标 | 对比）+ 全局队列入口（图标+数量）+ 设置。  
  - **主内容区**：按场景切换，每个场景内再分「文件区 / 格式与预设 / 参数 / 执行」。  
  - **底部或侧边**：常驻迷你队列（当前任务 + 进度），点击展开完整队列。

- **转视频场景示例**：  
  - 左：拖拽区 + 已选文件列表（缩略图 + 文件名 + 移除）。  
  - 中上：目标格式（大卡片或下拉）+ 预设快捷按钮（如「Web 高质量」「存档」）。  
  - 中下：高级参数（折叠面板，默认收起的如像素格式、自定义参数）。  
  - 右或底部：当前任务进度 + 「加入队列」主按钮。

### 5.2 交互原则

- **拖拽优先**：支持把文件/文件夹拖入窗口即可加入列表，路径输入作为补充。  
- **预设前置**：常用格式与质量用大按钮或卡片，点击后再展示或微调高级项。  
- **进度可见**：队列中每项显示进度条、状态、输出路径；完成后可「在 Finder 中显示」。  
- **容错与反馈**：格式不支持、路径无权限、FFmpeg 未安装等，用 Toast 或内联提示说明原因与解决方式。

### 5.3 视觉规范建议

- **色彩**：保留深色主色，增加「主操作色」（如绿色/蓝色）用于主按钮、进度、成功状态；危险操作用红色。  
- **字阶**：标题 / 区块标题 / 正文 / 辅助文字 至少 3 级，保证层级。  
- **间距与圆角**：统一 4/8/16/24px 等基础间距，卡片与按钮统一圆角（如 8px）。  
- **图标**：场景入口、队列、设置、格式类型等用统一风格图标（可选用 SF Symbols / Phosphor / Lucide 等），便于扫视。

### 5.4 响应式与桌面优先

- 以**桌面端**为主（1280×800 及以上），布局可两栏或三栏。  
- 若保留浏览器访问，小屏时可改为单栏、导航收起到汉堡菜单。
