# Fluxio 实际上线操作教程

> 面向不懂开发、不懂服务器的个人开发者，可直接照着做。

**版本约定**：当前仓库发版目标为 **v1.1.1**（Fluxio UI 细节迭代）；曾以 **v1.1.0** 为上一线上基线、**v1.0.0** 为初始生产基线。详细变更与 Git 约定见 **[发版笔记.md](./发版笔记.md)**。

**本文默认流程**：**香港地域**云服务器 + **免 ICP 备案**，域名解析到香港机器即可对外提供 HTTP/HTTPS。若你使用**中国大陆**服务器，需另按云厂商要求完成**备案**，见下文「大陆与香港」对照。

---

## 第一部分：一句话结论

1. **当前项目已经具备 MVP 上线条件**，可直接部署。
2. **你需要购买**：一个域名（如 `yourname.com`）+ 一台 Ubuntu 云服务器（推荐 **香港地域**，2 核 4GB、40GB 硬盘）。
3. **上线方式**：单机部署，域名 **A 记录** 指向服务器公网 IP，通过主路径 `https://你的域名.com/` 访问；上传限制 200MB，temp / logs 会自动清理。
4. **香港服务器**：**无需**在中国大陆进行 ICP 备案即可用域名对外访问（仍需在云厂商控制台**放行 80/443**，并完成下面 Nginx / HTTPS 步骤）。

## 第三部分：实际上线操作教程

---

### 阶段 A：购买资源

#### 1. 先买域名

- **去哪里买**：阿里云（万网）、腾讯云、华为云、Cloudflare、GoDaddy 等，任选其一。
- **买什么**：`.com` / `.net` / `.help` 等均可；若走**香港免备案**流程，**不要**把域名强制绑死在「仅大陆备案后才能解析」的套餐上（以注册商说明为准）。
- **买完要做什么**：先不用解析，等服务器买好并拿到**公网 IP** 后再添加 **A 记录**。

#### 2. 买服务器（推荐：香港）

- **买什么类型**：轻量应用服务器（或云服务器 CVM / ECS），**地域选「香港」**（各云厂商名称可能为「中国香港」「Hong Kong」）。
- **推荐平台**：阿里云、腾讯云、华为云、AWS、Vultr 等（只要提供 **香港机房 + 公网 IP** 即可）。
- **配置建议**：2 核 CPU、4GB 内存、40GB 系统盘、带宽按预算 3Mbps 起。
- **系统选择**：**Ubuntu 22.04 LTS**（或 24.04 LTS）。
- **为何选香港**：在**不做中国大陆 ICP 备案**的前提下，可用**域名 + HTTPS** 正常对外建站；大陆用户访问延迟通常略高于华东，但一般可接受。

#### 3. 云厂商控制台：放行端口（必做）

购买后立刻在控制台找到 **防火墙 / 安全组**（名称因厂商而异）：

- 放行 **TCP 22**（SSH 登录）
- 放行 **TCP 80**（HTTP）
- 放行 **TCP 443**（HTTPS，配置证书后需要）

否则浏览器会出现「连接被拒绝」或长时间打不开。

#### 4. 备案与地域：大陆 vs 香港（对照）

| 项目 | 中国大陆服务器 | 香港服务器（本文默认） |
|------|----------------|------------------------|
| ICP 备案 | 使用大陆机房对外提供网站，**通常必须备案** | **无需**大陆 ICP 备案即可绑定域名访问 |
| 访问与合规 | 需按云厂商与法规完成备案、接入 | 按服务商条款使用；面向大陆用户时自行评估业务合规 |
| GitHub `git clone` | 偶发超时，可用本机 `rsync`/`scp` 同步代码代替 | 一般较稳定，仍建议保留 `rsync` 作为备用 |

若你已有**大陆轻量 / ECS**且已备案，可按本教程其余步骤同样部署，仅多一步**备案域名与接入**，并注意部分网络环境下需完成**域名在平台的备案关联**后才能用 80/443。

### 阶段 B：服务器初始化

#### 1. 登录服务器

买完服务器后，控制台会给你：

- **公网 IP**：例如 `123.45.67.89`
- **用户名**：可能是 `root`，也可能是 `ubuntu`、`admin`（以厂商文档为准）
- **密码**或 **SSH 密钥**

用终端登录（二选一，用户名按控制台为准）：

```bash
ssh root@你的公网IP
# 或
ssh ubuntu@你的公网IP
```

首次登录会提示是否信任，输入 `yes`，再输入密码。

**权限说明（全文通用）**：下列命令按 **Ubuntu 普通用户 + sudo** 编写（如 `ubuntu`）。若你使用 **`root` 登录**，可将命令里的 `sudo` 全部省略后执行，效果相同。

#### 2. 登录后第一步：更新系统

```bash
sudo apt update && sudo apt upgrade -y
```

#### 3. 安装 Node.js 20

```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs
node -v
npm -v
```

看到 `v20.x` 和 npm 版本号即表示安装成功。

#### 4. 安装 FFmpeg

```bash
sudo apt install -y ffmpeg
ffmpeg -version
```

能看到版本信息即表示成功。

#### 5. 安装 Nginx

```bash
sudo apt install -y nginx
sudo systemctl enable nginx
```

#### 6. 安装 PM2（进程守护）

全局安装需管理员权限：

```bash
sudo npm install -g pm2
```

### 阶段 C：部署项目

#### 1. 项目放到哪里

放到：`/var/fluxio/app`

在服务器上先创建目录并**把归属设给当前登录用户**（否则后面 `npm ci`、PM2 可能权限不足）：

```bash
sudo mkdir -p /var/fluxio/app
sudo chown -R "$(whoami):$(whoami)" /var/fluxio
```

#### 2. 如何上传项目

**方式一：本机用 SCP 上传（推荐）**

在你自己的电脑上（不是服务器）打开终端，进入项目目录，执行：

```bash
cd /Users/n7/cursor/ffmpeg-tool
scp -r ./* ubuntu@你的公网IP:/var/fluxio/app/
```

把 `ubuntu` 换成你 SSH 登录用的**用户名**（控制台常见为 `ubuntu`、`admin`、`root`），把 `你的公网IP` 换成真实 IP，按提示输入密码。

若你用 **`root@`** 上传，上传后文件属主为 root，请在服务器执行一次（把 `ubuntu` 改成你实际用来跑 PM2 的用户名）：

```bash
sudo chown -R ubuntu:ubuntu /var/fluxio/app
```

**需要排除 node_modules 和 logs 时**（可选，减少上传量）：

```bash
rsync -avz --exclude 'node_modules' --exclude 'logs' --exclude 'temp' . ubuntu@你的公网IP:/var/fluxio/app/
```

**方式二：用 Git（如果项目在 GitHub 等）**

**必须先有目录 `/var/fluxio/app`**。若你**还没执行**上面「1. 项目放到哪里」里的两行 `mkdir` / `chown`，请用下面**整段**一次执行（将仓库地址换成你的；示例为 Fluxio 仓库）：

```bash
sudo mkdir -p /var/fluxio/app
sudo chown -R "$(whoami):$(whoami)" /var/fluxio
sudo apt install -y git
cd /var/fluxio/app
git clone https://github.com/N7FAA/fluxio.git .
npm ci --omit=dev
```

若你**已经**执行过「1」里的 `mkdir` 与 `chown`，只需从 `sudo apt install -y git` 开始执行到 `npm ci` 即可。上面整段已包含 `npm ci` 时，**跳过**下面「3」。

若 `git clone` / `git pull` 报 TLS 超时或中断，可改用**方式一**从本机 `rsync` 同步；香港机房通常较少出现该问题。

**常见错误**：未建目录就 `cd /var/fluxio/app`，会报 `No such file or directory`；若此时在**家目录**误执行了 `git clone ... .`，请先 `cd /var/fluxio/app`（建好目录后）再克隆，或删掉错误目录里的 `.git` 后重试。

#### 3. 若仅用 SCP/rsync 上传（未用上面 Git 整段）

在服务器上执行：

```bash
cd /var/fluxio/app
npm ci --omit=dev
```

#### 4. 是否需要构建

**不需要。** 当前项目是纯 Node + 静态文件，没有前端构建步骤。

#### 5. 配置环境变量

项目已提供 `ecosystem.config.cjs`，默认已包含：

- `NODE_ENV=production`
- `PORT=4000`
- `FFMPEG_ROOT=/var/fluxio/app`
- `MAX_FILE_SIZE=209715200`（200MB）

一般无需修改，除非你要改端口或数据目录。

#### 6. 启动项目

```bash
cd /var/fluxio/app
pm2 start ecosystem.config.cjs
```

#### 7. 设置开机自启

```bash
pm2 startup
```

终端会输出一行 `sudo env PATH=...` 的命令，**原样复制执行**，再执行：

```bash
pm2 save
```

#### 8. 确认项目已运行

```bash
pm2 status
curl -s http://127.0.0.1:4000/ | head -20
```

能看到 HTML 内容说明服务正常。

### 阶段 D：配置网站访问

#### 1. Nginx 配置

在服务器上执行（文件名可自定，与下面 `ln` 一致即可）：

```bash
sudo nano /etc/nginx/sites-available/fluxio
```

将下面内容粘贴进去，并做三处替换：

- `your-domain.com` → 你的根域名（如 `example.com`）
- `www.your-domain.com` → 你的 `www` 子域名（如 `www.example.com`）
- `你的公网IP` → 控制台显示的**服务器公网 IP**（用于 `http://IP` 直接访问时不出现 403）

```nginx
server {
    listen 80;
    # 必须同时写：域名 + www + 公网 IP，否则用浏览器访问 http://IP 时 Host 不匹配，可能落到默认站点得到 403
    server_name your-domain.com www.your-domain.com 你的公网IP;

    client_max_body_size 200M;

    location / {
        proxy_pass http://127.0.0.1:4000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        proxy_read_timeout 600s;
        proxy_connect_timeout 600s;
        proxy_send_timeout 600s;
    }
}
```

保存后（Ctrl+O 回车，Ctrl+X 退出），执行：

```bash
sudo ln -sf /etc/nginx/sites-available/fluxio /etc/nginx/sites-enabled/
sudo rm -f /etc/nginx/sites-enabled/default
sudo nginx -t
sudo systemctl reload nginx
```

`nginx -t` 显示 `syntax is ok` 即表示配置正确。

**若暂时没有域名**：可把 `server_name` 写成 `server_name _ 你的公网IP;` 或仅 `server_name 你的公网IP;`，仅用 IP 访问；有域名后再改回并 `reload`。

#### 2. 域名解析

在域名购买商的控制台里：

- 添加 **A 记录**
- 主机记录：`@`（代表根域名）或 `fluxio`（代表 fluxio.yourname.com）
- 记录值：填你的服务器公网 IP
- 保存后等待几分钟到几十分钟生效

#### 3. 配置 HTTPS（建议）

将 `你的域名.com` 换成已解析到本机的域名（可先完成阶段 D 的域名解析再执行）：

```bash
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d 你的域名.com
```

若同时为 `www` 子域名申请证书，且 DNS 已指向本机，可追加参数：`-d www.你的域名.com`。按提示输入邮箱、同意条款，完成后会自动配置 HTTPS。证书到期前会自动续期。

**若暂时用 IP 访问**：跳过此步，IP 无法申请 Let's Encrypt 证书。

#### 4. 防火墙（系统层）

云厂商**安全组 / 防火墙**已在阶段 A 放行 80/443/22 后，再在系统内执行（可选，按需）：

```bash
sudo ufw allow 80
sudo ufw allow 443
sudo ufw allow 22
sudo ufw enable
```

#### 5. 主路径 `/` 和 200MB 限制确认

- 在服务器上：`curl -sI http://127.0.0.1/` 应为 **200**；`pm2 status` 中 `fluxio` 为 **online**。
- 在你自己电脑或手机（**非公司受限网络**）：访问 `http://你的公网IP/` 与 `http://你的域名/`，应看到 Fluxio 首页。
- 若本机始终失败、手机 4G 正常，多为公司网络或终端安全软件（如「云壳」）拦截，需在对应软件中加白域名/IP，或换网络测试。
- 页面上传提示应写有「单文件不超过 200MB」；Nginx 中 `client_max_body_size 200M` 已保证上传上限。

### 阶段 F：上线后日常维护

作为不懂技术的个人开发者，日常只需做这些：

**每天**：
- 打开网站看一眼能否正常访问（约 1 分钟）。

**每周**：
- SSH 登录服务器，执行 `pm2 status`，确认进程在运行。
- 如有 `pm2 logs fluxio` 中的明显报错，可截图发给懂技术的朋友帮忙看。

**常用命令**（在服务器上执行）：

```bash
pm2 status        # 查看是否在运行
pm2 logs fluxio   # 查看最近日志
pm2 restart fluxio  # 若有异常，可尝试重启
```

**需要立刻处理的情况**：
- 网站打不开、一直转圈。
- 上传或转换一直失败。
- 服务器控制台显示关机、欠费、被封禁等。

### 阶段 G：以后怎么更新版本

#### 1. 本地改完后

在你自己电脑上完成修改并测试通过。

#### 2. 更新到服务器

**方式一：用 SCP 重新上传（简单）**

```bash
cd /Users/n7/cursor/ffmpeg-tool
scp -r ./* ubuntu@你的公网IP:/var/fluxio/app/
```

`ubuntu` 与 IP 替换规则同阶段 C。再在服务器上执行（见下）。

**方式二：用 Git**

先 SSH 登录服务器（用户名按你的实例为准）：

```bash
ssh ubuntu@你的公网IP
```

登录后在**服务器**上执行：

```bash
cd /var/fluxio/app
git pull origin main
npm ci --omit=dev
pm2 restart fluxio
```

将 `main` 换成你仓库的默认分支名（如有不同）。

#### 3. 更新前备份（建议）

在服务器上（备份目录与 `app` 并列）：

```bash
cd /var/fluxio
cp -r app "app_backup_$(date +%Y%m%d)"
```

若提示权限不足，可改为：`sudo cp -r app "app_backup_$(date +%Y%m%d)"`，之后用 `sudo chown -R "$(whoami):$(whoami)" app_backup_*` 调回属主以便操作。

#### 4. 更新后重启

```bash
cd /var/fluxio/app
pm2 restart fluxio
```

#### 5. 验证是否成功

打开 `https://你的域名.com/`，确认新功能或修复已生效。

#### 6. 若有问题如何回滚

把 `YYYYMMDD` 换成你备份目录名的日期（例如 `app_backup_20260402` 里的 `20260402`）：

```bash
cd /var/fluxio
rm -rf app
mv "app_backup_YYYYMMDD" app
cd app
pm2 restart fluxio
```

若 `rm`/`mv` 提示权限不足，在每条前加 `sudo`，并在 `pm2 restart` 前执行 `sudo chown -R "$(whoami):$(whoami)" /var/fluxio/app`。

## 附录二：常见问题速查

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| 浏览器「连接被拒绝」 | 安全组未放行 80/443 | 云控制台放行 TCP 80、443 |
| `http://公网IP/` 返回 **403**，但 `curl 127.0.0.1` 为 200 | `server_name` 未包含该 **IP** | 按阶段 D 把公网 IP 写入 `server_name` 并 `reload` |
| 公司电脑无法访问、手机 4G 正常 | 公司终端安全 / 代理拦截 | 换网络或在安全软件中对域名/IP **加白**（含浏览器进程） |
| `git pull` 失败 | 网络不稳定 | 本机 `rsync` 同步到 `/var/fluxio/app` 后执行 `npm ci` + `pm2 restart` |
| 查看 Nginx 错误日志 | 排查 502/504 | `sudo tail -n 80 /var/log/nginx/error.log` |
