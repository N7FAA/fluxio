## 第三部分：实际上线操作教程

---

### 阶段 A：购买资源

#### 1. 先买域名

- **去哪里买**：阿里云（万网）、腾讯云、华为云、GoDaddy 等，任选其一。
- **买什么**：`.com` 或 `.cn`，例如 `fluxio.yourname.com` 或 `yourname.com`。
- **买完要做什么**：先不用解析，等服务器买好并拿到公网 IP 后再解析。

#### 2. 买服务器

- **买什么类型**：轻量应用服务器（或云服务器 CVM）。
- **推荐平台**（国内）：阿里云、腾讯云、华为云。
- **配置建议**：2 核 CPU、4GB 内存、40GB 系统盘、带宽 3–5Mbps。
- **系统选择**：Ubuntu 22.04 LTS（或 20.04）。
- **地域**：选离你主要用户近的，如华北、华东。

#### 3. 备案（仅国内服务器需要）

- 使用国内服务器对外提供网站，域名通常需要备案。
- 备案流程在云厂商控制台里有引导，按提示提交资料即可。
- **如果暂时不想备案**：可以先用服务器 IP 直接访问做测试；正式上线再用备案域名。

---

### 阶段 B：服务器初始化

#### 1. 登录服务器

买完服务器后，控制台会给你：

- **公网 IP**：例如 `123.45.67.89`
- **root 密码**（或密钥）

用终端登录：

```bash
ssh root@你的公网IP
```

首次登录会提示是否信任，输入 `yes`，再输入密码。

#### 2. 登录后第一步：更新系统

复制执行：

```bash
apt update && apt upgrade -y
```

#### 3. 安装 Node.js 20

```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt install -y nodejs
node -v
npm -v
```

看到 `v20.x` 和 npm 版本号即表示安装成功。

#### 4. 安装 FFmpeg

```bash
apt install -y ffmpeg
ffmpeg -version
```

能看到版本信息即表示成功。

#### 5. 安装 Nginx

```bash
apt install -y nginx
systemctl enable nginx
```

#### 6. 安装 PM2（进程守护）

```bash
npm install -g pm2
```

---

### 阶段 C：部署项目

#### 1. 项目放到哪里

放到：`/var/fluxio/app`

先创建目录：

```bash
mkdir -p /var/fluxio/app
```

#### 2. 如何上传项目

**方式一：本机用 SCP 上传（推荐）**

在你自己的电脑上（不是服务器）打开终端，进入项目目录，执行：

```bash
cd /Users/n7/cursor/ffmpeg-tool
scp -r ./* root@你的公网IP:/var/fluxio/app/
```

把 `你的公网IP` 换成你的真实 IP，按提示输入密码。

**需要排除 node_modules 和 logs 时**（可选，减少上传量）：

```bash
rsync -avz --exclude 'node_modules' --exclude 'logs' --exclude 'temp' . root@你的公网IP:/var/fluxio/app/
```

**方式二：用 Git（如果项目在 GitHub 等）**

在服务器上执行：

```bash
apt install -y git
cd /var/fluxio/app
git clone 你的仓库地址 .
```

#### 3. 上传后执行的命令

在服务器上执行：

```bash
cd /var/fluxio/app
npm ci --omit=dev
```

#### 4. 是否需要构建

**不需要。** 当前项目是纯 Node + 静态文件，没有前端构建步骤。

#### 5. 配置环境变量

项目已提供 `ecosystem.config.cjs`，默认已包含：

- `NODE_ENV=production`
- `PORT=4000`
- `FFMPEG_ROOT=/var/fluxio/app`
- `MAX_FILE_SIZE=209715200`（200MB）

一般无需修改，除非你要改端口或数据目录。

#### 6. 启动项目

```bash
cd /var/fluxio/app
pm2 start ecosystem.config.cjs
```

#### 7. 设置开机自启

```bash
pm2 startup
```

终端会输出一行 `sudo env PATH=...` 的命令，**原样复制执行**，再执行：

```bash
pm2 save
```

#### 8. 确认项目已运行

```bash
pm2 status
curl -s http://127.0.0.1:4000/ | head -20
```

能看到 HTML 内容说明服务正常。

### 阶段 D：配置网站访问

#### 1. Nginx 配置

在服务器上执行：

```bash
nano /etc/nginx/sites-available/fluxio
```

将下面内容粘贴进去（注意把 `your-domain.com` 改成你的域名或留空用 IP）：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    client_max_body_size 200M;

    location / {
        proxy_pass http://127.0.0.1:4000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        proxy_read_timeout 600s;
        proxy_connect_timeout 600s;
        proxy_send_timeout 600s;
    }
}
```

保存后（Ctrl+O 回车，Ctrl+X 退出），执行：

```bash
ln -sf /etc/nginx/sites-available/fluxio /etc/nginx/sites-enabled/
nginx -t
systemctl reload nginx
```

`nginx -t` 显示 `syntax is ok` 即表示配置正确。

**如果暂时没有域名**，把 `server_name your-domain.com;` 改成 `server_name _;` 或直接删除这一行，用 IP 访问。

#### 2. 域名解析

在域名购买商的控制台里：

- 添加 **A 记录**
- 主机记录：`@`（代表根域名）或 `fluxio`（代表 fluxio.yourname.com）
- 记录值：填你的服务器公网 IP
- 保存后等待几分钟到几十分钟生效

#### 3. 配置 HTTPS（建议）

```bash
apt install -y certbot python3-certbot-nginx
certbot --nginx -d 你的域名.com
```

按提示输入邮箱、同意条款，完成后会自动配置 HTTPS。证书到期前会自动续期。

**若暂时用 IP 访问**：跳过此步，IP 无法申请 Let's Encrypt 证书。

#### 4. 防火墙（如有）

```bash
ufw allow 80
ufw allow 443
ufw allow 22
ufw enable
```

#### 5. 主路径 `/` 和 200MB 限制确认

- 访问 `http://你的域名或IP/`，应看到 Fluxio 首页。
- 页面上传提示应写有「单文件不超过 200MB」。
- Nginx 配置中 `client_max_body_size 200M` 已保证 200MB 上传。

### 阶段 E：上线前检查

按下面逐项核对：

| # | 检查项 | 如何验证 |
|---|--------|----------|
| 1 | 页面能否打开 | 浏览器访问 `https://你的域名.com/`，看到 Fluxio 首页 |
| 2 | 上传是否正常 | 上传一个小图（如 1MB PNG），能成功加入列表 |
| 3 | 超过 200MB 是否提示正确 | 上传一个大于 200MB 的文件，应提示超限 |
| 4 | 转换是否正常 | 上传后点击「开始转换」，等待完成 |
| 5 | 任务列表是否正常 | 转换过程中和完成后，任务列表显示正确 |
| 6 | 文件详情是否正常 | 点击「查看」进入详情页，信息正确 |
| 7 | 下载是否正常 | 点击「下载」能顺利下载结果文件 |
| 8 | temp / logs 是否正常 | 项目会自动清理，可通过 `ls -la /var/fluxio/app/temp/uploads` 查看 |
| 9 | HTTPS 是否正常 | 地址栏显示锁形图标，无不安全提示 |

### 阶段 F：上线后日常维护

作为不懂技术的个人开发者，日常只需做这些：

**每天**：
- 打开网站看一眼能否正常访问（约 1 分钟）。

**每周**：
- SSH 登录服务器，执行 `pm2 status`，确认进程在运行。
- 如有 `pm2 logs fluxio` 中的明显报错，可截图发给懂技术的朋友帮忙看。

**常用命令**（在服务器上执行）：

```bash
pm2 status        # 查看是否在运行
pm2 logs fluxio   # 查看最近日志
pm2 restart fluxio  # 若有异常，可尝试重启
```

**需要立刻处理的情况**：
- 网站打不开、一直转圈。
- 上传或转换一直失败。
- 服务器控制台显示关机、欠费、被封禁等。

### 阶段 G：以后怎么更新版本

#### 1. 本地改完后

在你自己电脑上完成修改并测试通过。

#### 2. 更新到服务器

**方式一：用 SCP 重新上传（简单）**

```bash
cd /Users/n7/cursor/ffmpeg-tool
scp -r ./* root@你的公网IP:/var/fluxio/app/
```

再在服务器上执行（见下）。

**方式二：用 Git**

```bash
ssh root@你的公网IP
cd /var/fluxio/app
git pull
npm ci --omit=dev
pm2 restart fluxio
```

#### 3. 更新前备份（建议）

在服务器上：

```bash
cd /var/fluxio
cp -r app app_backup_$(date +%Y%m%d)
```

#### 4. 更新后重启

```bash
cd /var/fluxio/app
pm2 restart fluxio
```

#### 5. 验证是否成功

打开 `https://你的域名.com/`，确认新功能或修复已生效。

#### 6. 若有问题如何回滚

```bash
cd /var/fluxio
rm -rf app
mv app_backup_YYYYMMDD app
cd app
pm2 restart fluxio
```

把 `YYYYMMDD` 换成你备份时的日期。
