┌─────────────────────────────────────────────────────────────────┐
│  用户操作                                                         │

                            │

        ┌───────────────────┼───────────────────┐
        │                   │                   │
        ▼                   ▼                   ▼

## 三、实现方案

### 3.1 后端：最小改动

现有能力已覆盖主流程，**无需新增接口**：

| 能力 | 现有实现 | 说明 |
|------|----------|------|
| 上传 | `POST /api/upload` | 存 temp/uploads，返回 `{ path, name, size }` |
| 转码 | `POST /api/queue` | 接收 inputPath、container、params，输出到 outputDir |
| 任务状态 | `GET /api/jobs` | 含 outputPath、status、progress |
| 结果访问 | `app.use('/uploads', express.static(UPLOAD_DIR))` | 输出在 uploads 下，可直接 `/uploads/xxx.webm` 下载 |

**需要的小改动**：

1. **输出目录**：上传文件的转码任务，`outputDir` 固定为 `UPLOAD_DIR`（当前逻辑已是「默认同输入目录」，上传后 input 在 uploads，输出也会在 uploads）。
2. **可选**：若希望输出文件名更清晰，可在 queue 的 payload 中显式传 `outputDir: UPLOAD_DIR`（或通过 env 配置）。

**结论**：后端可零改动跑通，仅需前端按 Fluxio 参数格式调用现有 API。

---

### 3.2 前端：Fluxio 风格页面

新增独立页面 `fluxio.html` + `fluxio.js` + `fluxio.css`，与现有 index 并存，通过路由或入口切换。

**页面与交互规范**：详见 `FLUXIO-PRD.md`（基于 fluxio.pen 设计稿）

#### 3.2.1 页面结构

```
┌────────────────────────────────────────────────────────────┐
│  Fluxio · 让动效轻松落地                                    │
│  面向线上页面的开放转码工具                                 │
├────────────────────────────────────────────────────────────┤
│  [ 图片 ]  [ 动态图像 ]  [ 视频 ]     ← Tab                │
├────────────────────────────────────────────────────────────┤
│  ┌──────────────────────────────────────────────────────┐  │
│  │  拖拽或点击上传                                       │  │
│  │  支持 PNG / JPG / WEBP（图片 Tab）                     │  │
│  └──────────────────────────────────────────────────────┘  │
│                                                            │
│  文件列表：                                                 │
│  · xxx.png  1.2MB  1920×1080  [删除]                       │
│                                                            │
│  输出设置：                                                 │
│  格式 [WEBP ▼]  分辨率 [原始 ▼]  质量 [推荐 ▼]             │
│                                                            │
│  [ 开始转换 ]                                               │
├────────────────────────────────────────────────────────────┤
│  任务状态：处理中... 45%                                    │
│  结果：                                                    │
│  · xxx.webp  原始 1.2MB → 输出 320KB  压缩 73%  [查看][下载]│
└────────────────────────────────────────────────────────────┘
```

#### 3.2.2 数据流

| 步骤 | 前端行为 | 后端接口 |
|------|----------|----------|
| 1. 上传 | 用户选择/拖拽文件 → FormData → fetch POST /api/upload | 返回 `{ path, name, size }` |
| 2. 创建任务 | 收集 outputConfig，调用 POST /api/queue | 见下方参数映射 |
| 3. 轮询状态 | setInterval 调用 GET /api/jobs | 返回任务列表 |
| 4. 下载 | 任务 success 时，`outputPath` 在 uploads → 构造 `/uploads/${basename(outputPath)}`，用 a 标签 download |

#### 3.2.3 Fluxio 参数 → 现有 API 映射

**图片 Tab**

| Fluxio 配置 | 映射到 queue payload |
|-------------|----------------------|
| 格式 PNG/JPG/WEBP | `container`: png / jpg / webp |
| 分辨率 原始/0.75x/0.5x | `params.scale`: 不传 / `-1:540`(假设1080) / `-1:270`，或根据 ffprobe 动态计算 |
| 质量 推荐/体积/清晰 | `params.crf`: 30 / 35 / 23（PNG 用不同策略，见下） |

*注：静态图用 FFmpeg 转 PNG/JPG/WEBP 可行，但 Sharp 更优。MVP 阶段可先用 FFmpeg 统一处理。*

**动态图像 Tab**

| Fluxio 配置 | 映射到 queue payload |
|-------------|----------------------|
| 格式 GIF/APNG/WEBP | `container`: gif / apng / webp |
| 分辨率 | 同上 |
| 帧率 原始/30/24 | `params.frameRate`: 不传 / 30 / 24 |
| 质量 | `params.crf`: 30 / 35 / 23（webp）；GIF 用 palette 等 |

**视频 Tab**

| Fluxio 配置 | 映射到 queue payload |
|-------------|----------------------|
| 格式 MP4/WEBM | `container`: mp4 / webm |
| 分辨率、帧率、质量 | 同上 |

#### 3.2.4 序列帧 ZIP 规范

| 项目 | 要求 |
|------|------|
| 命名格式 | `前缀_编号`，如 `demo_0001.png` |
| 前缀 | 同一 ZIP 内需完全一致，支持字母、数字、下划线、连字符 |
| 编号 | 4–6 位纯数字，位数一致，从 0001 开始连续递增 |
| 格式 | PNG / JPG |
| 帧数 | ≤ 600 |
| 尺寸 | 所有帧宽高一致 |

### 3.3 路由与入口

**方案 A（推荐）**：新增 `/fluxio` 路由，返回 `fluxio.html`

```javascript
// server.js 在 app.use(express.static(...)) 之前添加
app.get('/fluxio', (_, res) => {
  res.sendFile(path.join(STATIC_DIR, 'fluxio.html'));
});
```

访问 `http://localhost:4000/fluxio` 即 Fluxio 页面。

**方案 B**：在现有 index 顶部加「Fluxio 模式」链接，跳转 `/fluxio`。

## 四、实施步骤（建议顺序）

### Step 1：最小可跑通（1～2 天）

1. 新建 `fluxio.html`，只做「动态图像」Tab，单文件。
2. 上传区：点击选择文件，调用 `/api/upload`，展示文件名。
3. 输出设置：格式选 WEBP，其余默认。
4. 「开始转换」：调用 `/api/queue`，轮询 `/api/jobs`，完成后展示下载链接（`/uploads/xxx.webp`）。
5. 验证：上传 MOV → 选 WEBP → 转换 → 下载。

### Step 2：补齐三个 Tab（1 天）

1. Tab 切换，按 Tab 更新上传区文案、允许格式。
2. 图片 Tab：PNG/JPG/WEBP 输入 → PNG/JPG/WEBP 输出。
3. 视频 Tab：同动态图像输入 → MP4/WEBM 输出。
4. 输出设置根据 Tab 动态显示（格式、分辨率、质量、帧率）。

### Step 3：参数映射与体验（1 天）

1. 分辨率 0.75x/0.5x：先 ffprobe 取尺寸，再算 scale。
2. 质量档位 → CRF 映射。
3. 拖拽上传、错误提示、加载状态。

### Step 4：结果区增强（0.5 天）

1. 展示原始大小、输出大小、压缩比。
2. 图片/动图预览（img/video 标签）、视频播放。

## 五、MVP 范围收缩（可选）

为更快跑通，可先做：

| 项目 | 全量 | MVP |
|------|------|-----|
| Tab | 图片+动态图像+视频 | 仅「动态图像」 |
| 输入 | 多格式 | MP4/MOV/WEBM + GIF/APNG/WEBP |
| 输出格式 | 多选 | 仅 WEBP、MP4 |
| 分辨率 | 原始/0.75/0.5 | 仅原始 |
| 质量 | 三档 | 仅推荐 |
| 序列帧 ZIP | 支持 | 不做 |
| 批量 | 多文件 | 单文件 |

先实现「动态图像 Tab + 单文件 + 输出 WEBP/MP4 + 原始分辨率 + 推荐质量」，约半天可跑通。
