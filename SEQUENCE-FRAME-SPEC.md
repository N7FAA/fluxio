# 序列帧 ZIP 规范

> Fluxio 序列帧上传与转换的命名与校验规则

---

## 一、命名规则

序列帧文件需按 **前缀_编号** 命名，例如：

```
demo_0001.png
demo_0002.png
demo_0003.png
...
```

### 1.1 格式要求

| 项目 | 要求 |
|------|------|
| 命名格式 | `前缀_编号.扩展名` |
| 前缀 | 支持字母、数字、中文、空格、下划线、连字符 |
| 同一 ZIP | 所有帧的前缀需**完全一致** |
| 编号 | 4–6 位纯数字，位数一致 |
| 起始 | 从 `0001` 开始连续递增，无缺号 |
| 扩展名 | PNG、JPG、JPEG |

### 1.2 示例

| 合法 | 非法 |
|------|------|
| `demo_0001.png` | `0001.png`（缺少前缀） |
| `scene_0010.jpg` | `demo_0001.png` + `other_0002.png`（前缀不一致） |
| `anim_00001.png`（5 位） | `demo_001.png` + `demo_0010.png`（位数不一致） |
| | `demo_0002.png`（缺少 0001） |

---

## 二、其他约束

| 项目 | 限制 |
|------|------|
| 帧数 | ≤ 600 |
| 尺寸 | 所有帧宽高一致 |
| 格式 | PNG / JPG |

---

## 三、技术实现

- **接口**：`POST /api/upload-sequence`
- **FFmpeg 输入**：`-framerate X -i 目录/前缀_%04d.png`
- **校验**：解压后按上述规则校验，失败则返回错误并清理临时文件
