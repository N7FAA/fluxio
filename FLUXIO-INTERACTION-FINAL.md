# 二、实现口径（必须严格遵守）

| 项目 | 规则 |
|------|------|
| **视频时长校验** | 全部由前端完成，使用 HTML5 video metadata 读取 duration，不依赖后端 ffprobe |
| **taskId** | 点击「开始转换」且校验通过后立即生成，格式：`task_${timestamp}_${random}` |
| **fileId** | 任务创建时按文件顺序生成，格式：`file_0`、`file_1`、`file_2`... |
| **sourceTab** | 任务创建时记录，使用 state + sessionStorage 存储 |
| **直接访问路由** | 若状态不存在，显示错误态，不做自动恢复 |
| **下载全部** | 全部完成后才可点击，当前版本采用逐个下载 |
| **回到首页** | 清空前端状态，保留任务来源 Tab |
| **Tab 切换确认** | 有文件时使用简单确认弹窗 |
| **失败态** | 仅做基础失败态，不做重试 |
| **文件详情** | 独立路由页面，非弹窗；时长/帧率按「有则显示，无则隐藏」 |

# 四、路由

| 路由 | 对应页面 | 说明 |
|------|----------|------|
| `/` | 首页（P01~P07） | 空态与工作台为局部状态切换，不新增路由 |
| `/tasks/:taskId` | 任务列表（P08/P09） | 进行中/已完成为同一路由状态切换 |
| `/tasks/:taskId/files/:fileId` | 文件详情（P10/P11/P12） | 独立路由，非弹窗 |

# 五、全局规则

## 5.1 Tab 规则

| 项目 | 规则 |
|------|------|
| Tab 含义 | 目标输出类型 |
| Tab 项 | 图片 / 动态图像 / 视频 |
| 默认选中 | 图片 |
| 切换 Tab（无文件） | 直接切换 |
| 切换 Tab（有文件） | 弹确认：「切换分类将清空当前已上传文件和设置，是否继续？」→ 确认后清空并切换 |
| 切换后 | 进入对应 Tab 空态 |

## 5.2 工作区规则（首页）

| 项目 | 规则 |
|------|------|
| 上传成功后 | 空态替换为工作台，不跳路由 |
| 删除最后 1 个文件 | 回到对应 Tab 空态 |
| 清空 | 回到对应 Tab 空态 |
| 添加文件 | 在当前工作台追加文件 |
| 处理中 | 不允许编辑，不允许返回工作台，必须通过「回到首页」开始新任务 |

## 5.3 sourceTab 存储

| 项目 | 规则 |
|------|------|
| 记录时机 | 点击「开始转换」创建任务时 |
| 取值 | `image` / `animated` / `video` |
| 存储 | state + sessionStorage |
| 回到首页 | 优先读取 sourceTab，返回对应空态（image→P01, animated→P02, video→P03） |
| 兜底 | 无法读取时默认 P01 |

# 六、ID 生成规则

## 6.1 taskId

| 项目 | 规则 |
|------|------|
| 生成时机 | 点击「开始转换」且校验通过后立即生成 |
| 格式 | `task_${timestamp}_${random}` |
| 示例 | `task_1720000000000_ab12cd` |

## 6.2 fileId

| 项目 | 规则 |
|------|------|
| 生成时机 | 任务创建时按文件顺序生成 |
| 格式 | `file_0`、`file_1`、`file_2`... |
| 对应关系 | 与任务列表中的单个文件项一一对应 |
| 路由示例 | `/tasks/task_xxx/files/file_0` |

# 七、视频时长校验

## 7.1 规则

| Tab | 时长限制 |
|-----|----------|
| 动态图像 | ≤ 20 秒 |
| 视频 | ≤ 60 秒 |

## 7.2 实现方式

1. 用户选择视频文件后、加入列表前校验
2. 使用 `URL.createObjectURL(file)` 创建临时地址
3. 创建隐藏 `video` 元素
4. 监听 `loadedmetadata`
5. 读取 `video.duration`
6. 根据当前 Tab 判断是否超限
7. 超限则提示错误，不加入列表
8. 校验完成后释放 object URL

## 7.3 错误文案

| 场景 | 文案 |
|------|------|
| 动态图像 Tab 超时长 | 动态图像输出仅支持 20 秒以内的视频 |
| 视频 Tab 超时长 | 当前仅支持 60 秒以内的视频 |
| 元数据读取失败 | 无法读取视频信息，请更换文件后重试 |

# 八、直接访问路由处理

## 8.1 任务列表 `/tasks/:taskId`

若 state / session 中找不到该任务：

- 显示错误态
- 不做自动恢复
- 文案：**当前任务不存在或已失效，请返回首页重新开始**
- 提供「返回首页」按钮

## 8.2 文件详情 `/tasks/:taskId/files/:fileId`

若找不到任务或文件：

- 显示错误态
- 不强制跳转
- 文案：**当前文件不存在或已失效，请返回任务列表或重新开始**
- 提供「返回任务列表」「返回首页」按钮

## 8.3 当前版本不实现

- URL 直达后的任务恢复
- 刷新后的任务恢复
- 从后端重新拉取任务详情

# 九、页面级交互

## 9.1 首页空态（P01/P02/P03）

| 触发 | 条件 | 系统反馈 | 下一状态 |
|------|------|----------|----------|
| 点击上传 | 选择合法文件 | 解析并展示文件卡片 | P05/P06/P07 |
| 拖入合法文件 | 文件进入上传区 | 高亮上传框 | P04 |
| 拖入非法文件 | 格式不匹配 | 错误提示 | 当前空态 |

## 9.2 首页拖拽中（P04）

| 元素 | 状态 |
|------|------|
| 上传框边框 | 高亮 |
| 文案 | “释放以上传文件” |

| 触发 | 条件 | 系统反馈 | 下一状态 |
|------|------|----------|----------|
| 拖离 | 无 | 恢复默认 | P01/P02/P03 |
| 释放合法文件 | 当前 Tab 接受 | 执行上传 | P05/P06/P07 |
| 释放非法文件 | 当前 Tab 不接受 | 错误提示 | 当前空态 |

## 9.3 工作台（P05/P06/P07）

| 触发 | 条件 | 系统反馈 | 下一状态 |
|------|------|----------|----------|
| 点击删除 | 至少剩余 1 个 | 删除该文件 | 当前工作台 |
| 点击删除 | 删除后为空 | 清空工作区 | P01/P02/P03 |
| 点击清空 | 有文件 | 清空全部 | P01/P02/P03 |
| 点击添加文件 | 选择合法文件 | 追加到列表 | 当前工作台 |
| 点击开始转换 | 校验通过 | 生成 taskId，跳转 | P08 |

## 9.4 任务列表（P08/P09）

| 触发 | 条件 | 系统反馈 | 下一状态 |
|------|------|----------|----------|
| 点击查看 | 该文件已完成 | 跳转详情 | P10/P11/P12 |
| 点击下载 | 该文件已完成 | 下载单文件 | 当前 |
| 点击下载全部 | 全部完成 | 逐个下载 | 当前 |
| 点击下载全部 | 未全部完成 | 禁用 | - |
| 点击回到首页 | 任意时刻 | 清空状态，返回首页 | P01/P02/P03（保留 sourceTab） |

## 9.5 文件详情（P10/P11/P12）

| 触发 | 条件 | 系统反馈 | 下一状态 |
|------|------|----------|----------|
| 点击返回 | 无 | 返回任务列表 | P08/P09 |

**文件信息字段**：文件名、格式、尺寸、大小；动图/视频显示时长；动图显示帧率。时长/帧率按「有则显示，无则隐藏」。

# 十、质量滑块（QualitySlider）

| 项目 | 规则 |
|------|------|
| **档位** | 5 档：0%、25%、50%、75%、100% |
| **透出文案** | 仅 3 项：体积优先（0%）、推荐（50%）、清晰优先（100%） |
| **25% / 75%** | 不显示「偏小」「偏清晰」等文案，仅支持拖拽至该位置 |
| **默认值** | 推荐（50%） |
| **交互** | 可点击档位或拖动滑块；拖拽可停在任意档位（含 25%、75%） |
| **技术实现** | 后端需支持 5 档的 CRF/参数映射，见 FLUXIO-FEASIBILITY 质量档位 |

# 十三、状态流转图

```
/ （首页）
├─ 空态 P01/P02/P03
│   ├─ 切换 Tab（无文件）→ 对应空态
│   ├─ 切换 Tab（有文件）→ 确认 → 清空 → 对应空态
│   ├─ 拖入 → P04
│   └─ 上传成功 → P05/P06/P07（局部切换）
├─ 拖拽中 P04
│   ├─ 拖离 → 当前空态
│   └─ 释放合法文件 → P05/P06/P07
└─ 工作台 P05/P06/P07
    ├─ 删除最后/清空 → 当前空态
    └─ 开始转换 → /tasks/:taskId

/tasks/:taskId
├─ P08 进行中 → 全部完成 → P09 已完成
├─ 查看 → /tasks/:taskId/files/:fileId
└─ 回到首页 → /（保留 sourceTab）

/tasks/:taskId/files/:fileId
└─ 返回 → /tasks/:taskId
```

# 十四、给 Cursor 的实现指令

以下为 Fluxio 当前版本最终实现口径，请严格遵守：

1. **视频时长校验**：全部由前端完成，使用 HTML5 video metadata 读取 duration
2. **taskId**：点击「开始转换」且校验通过后立即生成，格式 `task_${timestamp}_${random}`
3. **fileId**：任务创建时按文件顺序生成，格式 `file_0`、`file_1`...
4. **sourceTab**：任务创建时记录，使用 state + sessionStorage 存储
5. **直接访问路由**：若状态不存在，显示错误态，不做自动恢复
6. **下载全部**：全部完成后才可点击，当前版本逐个下载
7. **回到首页**：清空前端状态，保留任务来源 Tab
8. **Tab 切换确认**：有文件时使用简单确认弹窗
9. **失败态**：仅做基础失败态，不做重试
10. **文件详情**：独立路由页面，非弹窗；时长/帧率按「有则显示，无则隐藏」
